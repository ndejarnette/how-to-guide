Making API calls

Let's see what kinds of calls we can make using our API

API Request Structure

With our Steam API, there are two different ways to submit requests: requests which include our API key and requests that do not require our API key. We will first examine a call that does not require our API key.

http://api.steampowered.com/ISteamNews/GetNewsForApp/v0002/?appid=377160&count=3&maxlength=300&format=json

Now, if you click on that link, you will see a bunch of data that probably means nothing to you right now. We will get to what all that means in the next section, but for now, let's break down this request.

http://api.steampowered.com/ This is the base url that all of our API calls will contain no matter what. This is where our request is being sent.
ISteamNews/ This portion is the "interface name" of our request. The interface contains methods which we also need to include next. The complete list of interfaces (and methods) can be found in the official documentation here.
GetNewsForApp/ This is the "method name" which describes what this request is ultimately doing. We can see that this method will help us get recent news articles for a particular game.
v0002/ This is the version attribute. You must reference the official documentation to include the correct version in your request.
?appid=377160 The appid here refers to the ID of the game that we want to get news for. For our example, the game that has id 377160 is Fallout 4 (great game btw). You can search for your favorite game's id here.
count=3 Count here refers to the number of news articles that we want returned.
maxlength=300 Maxlength is the maximum length of each news entry that we want returned. In this case it is 300 characters.
format=json Format is the output format that we want our results to be in. By default, if you do not specify this value, results will be returned in JSON format (more on this later).

As you can see, once we break everything down, putting together an API request isn't so bad after all. Now that we've analyzed a request that doeen's require a key, let's look at one that DOES require that API key that we signed up for. Consider the request below:

http://api.steampowered.com/ISteamUser/GetPlayerSummaries/v0002/?key=[PLACE KEY HERE]&steamids=76561197960435530

Now if you click on the above link without replacing the [PLACE KEY HERE] with your API key, then you will receive an error telling you that access was denied. This is because we are trying to access somewhat sensitive/private information. Can you figure out what we are trying to access here from what you know about interface and method names? Let's analyze some of the newer parts of this request.

?key= In all calls where our API key is required, this piece of the request address will need to be filled in with our steam API key.
steamids= In this particular request, steamids refers to the ID of the user that we wish to get a summary of.

Now what?

Well, now that we understand the basic structure of a request using our API, how do we do something useful with it? So far, we have only been using our requests in the address bar of our browser. When we do this, the data we want is just output as one big jumble on the screen. So how can we begin to incorporate these API calls into some sort of useful application to get our player summary and game news data? That's where Javascript comes in.
